# ProgressoSEP3

Multi-layer .NET application built with separation of concerns and structured architecture.

## 🏗 Architecture

The solution follows a layered architecture:

- WebAPI – REST endpoints
- App – Business logic layer
- DataAccessClient – Data access abstraction
- Shared – Shared models and contracts
- Testing – Unit testing layer

## 🛠 Tech Stack

- .NET
- C#
- REST API
- Layered architecture
- Dependency Injection

## 🔍 Project Purpose

Academic semester project focused on structured backend design, maintainability and separation of responsibilities.

## 🚀 What I Learned

- Designing multi-layer applications
- Dependency injection patterns
- Data abstraction
- Clean code practices
